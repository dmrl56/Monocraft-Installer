# Contributing to Monocraft Font Tool for VS Code

Thank you for considering contributing! This project welcomes improvements, bugfixes, and documentation updates.

## How to Contribute

1. **Fork** the repository (if you don't have push rights)
2. **Create a branch** for your change (e.g., `feature/add-dark-mode` or `fix/font-install-bug`)
3. **Make your changes**
   - Keep code modular and well-documented
   - Add or update tests if relevant
   - Update documentation in `docs/` as needed
4. **Commit** with clear, descriptive messages
5. **Push** your branch to your fork (or the main repo if you have access)
6. **Open a Pull Request (PR)**
   - Go to the repo on GitHub and click "New Pull Request"
   - Select your branch as the source and the main branch as the target
   - Fill in a clear title and description
   - Reference related issues (e.g., `Fixes #123`)
   - Add screenshots for UI changes if possible
7. **Participate in review**
   - Respond to feedback and make follow-up commits as needed
   - Ensure CI checks pass
8. **Merge**
   - After approval and passing checks, your PR will be merged
   - Delete your branch if no longer needed

## Pull Request (PR) Best Practices
- Keep PRs focused and small
- Provide context: what problem it fixes, how it was tested
- Use descriptive branch names and commit messages
- Ensure tests and linting pass
- Reference issues and add screenshots for UI changes

## Code Style
- Follow the structure and conventions in the existing codebase
- Use clear, descriptive names for classes, methods, and variables
- Document public methods and complex logic

## Questions?
- Open an issue or discussion on the repository
- See `docs/DEVELOPER.md` for build and codebase details
